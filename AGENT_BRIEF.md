# Delta-Curator Implementation Brief (Final, Agent-Proof)
*Single-source specification for repository scaffold and population*

---

## 1. Mission & Invariants (Non-Negotiable)

**Delta-Curator** is an event-sourced novelty accumulation system. It ingests external data in batches, evaluates each item for novelty relative to a curated base using an explicit comparator, applies policy decisions (merge/append/hold/reject), and persists an append-only audit log as the sole source of truth.

**Immutable Invariants:**
- **I0 (Source of Truth):** The event log is append-only; all read models are derived and rebuildable.
- **I1 (Atomic Progress):** Source cursor advancement occurs exclusively within successful `Committer.commit()`.
- **I2 (Semantic Identity):** Events deduplicated via `event_id` (content-addressed semantic core). Forward-compatible schema changes must not alter `event_id`.
- **I3 (Determinism):** Replay with identical source state and code produces identical event sequences. Requires injectable `Clock` (frozen in tests).
- **I4 (Transparency):** Runner stages are referentially transparent; no hidden global state.

## 3. Canonicalization & Hashing (Authoritative)

**Canonical JSON function (single implementation used everywhere):**
- UTF-8 JSON
- Object keys sorted lexicographically
- No whitespace
- Arrays preserve order
- Numbers normalized (no trailing zeros, no `+0`)
- Booleans lowercase
- `null` explicit
- No implicit defaults

**Hashing rules:**
- `payload_hash = sha256(canonical_json(payload))`
- `event_id = sha256(canonical_string(event_type, source_item_id, semantic_core(payload)))`
- `semantic_core(payload)` includes **required fields only** from the first version of the schema.  
  Optional forward-compatible fields are excluded.
- **`observed_at` is excluded from `semantic_core`** (audit-only; does not affect identity).
- Header fields are **never** included in `event_id`.

```ts
type EventHeader = {
  event_type: string;          
  event_version: string;       
  
  event_id: string;            // sha256(type + source_item_id + semantic_core(payload))
  payload_hash: string;        
  hash_alg: "dc-v1-semcore";
  
  trace_id: string;            // Generated by Runner per batch
  source_id: string;           
  source_item_id: string;      
  
  candidate_seq: number;       // Ordering only, Runner-assigned per stage
  commit_id?: string;          
  observed_at: string;         // From injectable Clock (audit only)
};
````

**Rules:**

* **Dedup key:** `event_id` globally unique.
* **Ordering key:** `(trace_id, source_item_id, candidate_seq)`.
* Runner generates `trace_id`. Source never sets it.

## 5. Commit Key (Authoritative Formula)

`commitKey` MUST be deterministic:

```
commitKey = sha256(canonical_json({
  source_id,
  old_state,
  item_ids_sorted: sort(items.map(i => i.source_item_id)),
  hash_alg_commit: "dc-v1-commitkey"
}))
```

* `old_state` is canonicalized JSON.
* Sorting ensures batch order does not affect idempotency.

## 6. Plugin Contracts (Key Excerpts)

```ts
interface Source extends PluginMeta {

    items: RawDocObserved[];
```

## 7. SQLite Committer (v0)

Tables: `commits`, `events`, `artifacts`, `source_state`, `facet_index`, `fingerprint_index`, `hold_index`, `curated_docs`.

Rules:

* `event_id` is PRIMARY KEY.
* `commit_key` UNIQUE.
* Commit is atomic: insert commit → insert events (idempotent by `event_id`) → apply mutations → persist cursor.

---

## 8. Artifact Protocol (v0 Local FS)

Even in v0, simulate two-phase behavior:

* Phase 1: write artifact to local filesystem (`./artifacts/{commitKey}-{name}.bin`)
* Phase 2: SQLite commit
* Crash recovery test must simulate “artifact written, DB not committed” and verify idempotent retry.

---

## 9. Read Model Rebuild Contract

Read models MUST be rebuildable from the `events` table alone:

* `rebuildToHead(log)` drops read model tables and replays events to re-derive indexes.
* `catchUp(log, fromOffset)` applies only new events.

Do **not** rely on persisted mutations for rebuild correctness.

---

## 10. Runner Semantics

* Runner generates `trace_id`.
* Envelopes (`CandidateDoc`, `CandidateAnalysis`) are immutable.
* Each stage emits exactly one event per candidate with increasing `candidate_seq`.
* No stage reads mutable global state except via `BaseView` / `ResolveView` snapshots.

---

## 11. Testing Requirements (Mandatory)

* **Idempotency:** Same `commitKey` → same `commitId`; no duplicate `event_id`s.
* **Collision resistance:** Two items with identical payload but different `source_item_id` → different `event_id`s.
* **Determinism:** With frozen Clock, replay produces identical `event_id`s and `commitKey`s.
* **Crash recovery:** Simulate crash between artifact write and SQLite commit; retry must not duplicate state.
* **Rebuild:** Drop read models; rebuild from events; state must match pre-rebuild.
