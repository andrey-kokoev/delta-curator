# AGENTS.md — Delta-Curator

This file is the **only instruction source** for agents scaffolding and populating the `delta-curator` repository. Follow it exactly. Do not invent APIs, commands, or features not specified here.

---

## 0) Mission

Build **Delta-Curator**: an event-sourced novelty accumulation system for continuous intelligence monitoring.  
Primary runtime is **Cloudflare Workers**. Storage is **D1 (state)** + **R2 (artifacts)**.  
Ranking uses **Cloudflare Workers AI** (ingest-time) and **Cloudflare AI Search** (query-time).  
No UI. The system exposes HTTP endpoints only.

---

## 1) Non-Negotiable Invariants

- **I0 — Append-only log**: Event log is the sole source of truth. Read models are derived and rebuildable.
- **I1 — Atomic progress**: Source cursor advances only within a successful `Committer.commit()`.
- **I2 — Semantic identity**: Global dedup by `event_id` (semantic-core hash). Forward-compatible fields must not change `event_id`.
- **I3 — Determinism**: With identical inputs and code, replay produces identical event sequences. Implement injectable `Clock` (freeze in tests).
- **I4 — Referential transparency**: Pipeline stages are pure; no hidden global state.
- **I5 — Cloudflare-native**: Worker is primary runtime. Local dev uses `wrangler dev`. CLI is optional thin HTTP client only.

## 3) Event Identity & Ordering (Correctness Critical)

```ts
type EventHeader = {
  event_type: string;
  event_version: string;

  // Dual-hash identity
  event_id: string;       // Base64(sha256(event_type + source_item_id + semantic_core(payload)))
  payload_hash: string;   // Base64(sha256(full_canonical_payload))
  hash_alg: "dc-v1-semcore";

  trace_id: string;       // Generated by Runner per batch
  source_id: string;
  source_item_id: string;

  candidate_seq: number; // Ordering within candidate only
  commit_id?: string;
  observed_at: string;   // From injectable Clock
};
````

Rules:

* Dedup key = `event_id` only (global).
* Ordering = `(trace_id, source_item_id, candidate_seq)`.
* Semantic core excludes forward-compatible optional fields.
* Canonical JSON: sorted keys, no whitespace.

## 4) Plugin Contracts (Batch-Oriented)

```ts
interface PluginMeta {
  id: string;
  spec(): JSONSchema;
  inputVersions(): Record<string, string>;
  outputVersions(): Record<string, string>;
}

interface Source extends PluginMeta {
  readBatch(state: SourceState, maxItems: number): Promise<{
    items: RawDocObserved[];
    newState: SourceState;   // proposed; persisted only on commit
  } | null>;
  acknowledge?(batchId: string): Promise<void>;
}

interface Normalizer extends PluginMeta {
  canHandle(mime: string): boolean;
  normalize(doc: CandidateDoc): Promise<DocNormalized>;
}

interface Extractor extends PluginMeta {
  extract(doc: CandidateDoc): Promise<Facet[]>;
}

interface Resolver extends PluginMeta {
  resolve(facets: Facet[], view: ResolveView): Promise<EntityResolution[]>;
}

interface Comparator extends PluginMeta {
  init(snapshot: Uint8Array | null): Promise<{ determinism: "deterministic" | "artifact" }>;
  compare(input: { doc: CandidateDoc; analysis: CandidateAnalysis }, baseView: BaseView):
    Promise<{ comparison: CandidateCompared; newSnapshot?: Uint8Array }>;
}

interface Decider extends PluginMeta {
  decide(input: { doc: CandidateDoc; analysis: CandidateAnalysis }):
    Promise<{ decision: Decision; proposedPatch?: Patch }>;
}

interface Merger extends PluginMeta {
  planMutations(patch: Patch, view: BaseView): Promise<ViewMutation[]>;
}
```

Decider is pure. Merger translates logical patches to physical mutations.

## 5) Storage & Commit Protocol (Cloudflare-Native)

* **D1Committer** applies mutations atomically with event persistence.
* **R2Artifacts** stores raw docs + comparator snapshots.
* Two-phase pattern: write artifacts to R2 (idempotent key), then commit to D1.
* `commitKey` is deterministic: `hash(sourceId + oldState + itemIds)`.

D1 schema (minimum): commits, events, artifacts, source_state, facet_index, fingerprint_index, hold_index, curated_docs. Index hold `(queue, until)`.

## 6) Worker API (Implement Exactly)

Routes:

* `POST /run` → run one batch `{ source_id, max_items, once }`
* `GET /inspect?since=PT24H&format=markdown`
* `GET /search?q=...&k=20&rerank=true`
* `GET /health`

Worker must implement `scheduled()` to call `/run` per config.

---

## 7) Cloudflare Ranking (Required)

Signals contract in `CandidateCompared.signals`:

* `rank_score: number`
* `rank_backend: "workers_ai_rerank" | "ai_search_rerank" | "none"`
* `rank_model?: string`

Backends:

* **Workers AI reranker** (ingest-time): model `@cf/baai/bge-reranker-base`. Deterministic passage construction (title + first N chars + facets summary).
* **AI Search rerank** (query-time): used by `/search`.

Ranking never affects novelty deltas in v0.

## 9) Tests (Must Pass)

* Idempotency: same `commitKey` → same `commitId`; no duplicate `event_id`.
* Collision resistance: identical payloads with different `source_item_id` → different `event_id`.
* Determinism: freeze Clock in tests.
* Crash recovery: artifact write succeeds, D1 commit fails → retry is idempotent.
* Worker E2E via miniflare/wrangler.

## 11) Definition of Done

* `wrangler dev` runs Worker locally.
* `/run` ingests deterministically and commits to D1.
* R2 artifacts written before commit; retries are idempotent.
* Workers AI rerank populates `rank_score`.
* `/inspect` returns markdown digest.
* `/search` returns reranked results when enabled.
* `pnpm test` passes (including idempotency + determinism).

Do not add UI, dashboards, topic orchestrators, claim-level KBs, or Railway services in v0/v1.
